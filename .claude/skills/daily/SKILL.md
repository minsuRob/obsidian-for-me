# /daily — 일일보고 자동 생성

기준 경로: `Work/Markhub-LLMWiki/` (이하 `WIKI/`). 대상 산출물: `WIKI/reports/daily/<YYYY-MM-DD>.md`.

## 처리 순서

### 1. 오늘 자료 수집
- 오늘 날짜를 확인한다.
- 오늘 `Processed: true`(또는 `처리일: 오늘`)가 붙어 `WIKI/raw/`로 이동된 원본들.
- 오늘 `최종수정`이 오늘인 `WIKI/wiki/**` 문서들(무엇이 바뀌었는지 히스토리/섹션 확인).
- 필요하면 `git -C <vault> diff --stat`으로 당일 변경 파일을 교차 확인.

### 2. 보고서 작성
`WIKI/reports/daily/<YYYY-MM-DD>.md`에 아래 형식으로 작성(이미 있으면 갱신):
```markdown
---
유형: daily
날짜: <YYYY-MM-DD>
담당: 나
최종수정: <YYYY-MM-DD>
태그: [일일보고]
---

# 일일보고 <YYYY-MM-DD>

## ✅ 오늘 한 일
- (처리한 회의/작업, 관련 [[문서]] 링크)

## 🚧 막힌 일 / 리스크
- (상태가 보류이거나 리스크로 적힌 항목)

## 📌 내일 할 일
- (열린 이슈·액션 아이템 중 우선순위 높은 것)

## 오늘 변경된 위키
- [[문서]] : 무엇이 어떻게 바뀜
```

### 3. 규칙
- 링크는 `[[문서명]]`으로 걸어 백링크가 형성되게 한다.
- 내용을 지어내지 않는다. 근거는 wiki/원본에서 가져온다.
- "막힌 일"은 `상태: 보류`이거나 원본에 리스크로 언급된 것 위주.
- 다 쓰면 파일 경로와 핵심 3줄 요약을 사용자에게 보고.
