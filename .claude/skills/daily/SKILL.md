# /daily — 일일보고 생성

Markhub-LLMWiki의 스탠드업형 일일보고를 만든다. 대상 폴더: `Work/Markhub-LLMWiki/20_Tracking/daily/`.

## 절차

1. **오늘 날짜 확인**: 시스템의 오늘 날짜를 `YYYY-MM-DD`로 잡는다(예: 2026-07-02). 파일 경로는 `Work/Markhub-LLMWiki/20_Tracking/daily/<날짜>.md`. 이미 있으면 덮지 말고 비어있는 섹션만 보완한다.

2. **어제 노트 이월**: `20_Tracking/daily/`에서 오늘 직전의 daily 노트를 찾는다. 그 노트의 **`### 🎯 오늘 할 일`** 중 완료되지 않은 항목을 오늘 노트의 **`### ✅ 어제 한 일`** 후보로 옮긴다(사용자가 완료/미완료를 다듬을 수 있게 링크는 유지).

3. **진행중 일감 반영**: `20_Tracking/tasks/`에서 `상태: 진행중`인 일감을 읽어 `### 🎯 오늘 할 일`과 `### 🔗 관련 일감`에 `[[일감명]]` 링크로 넣는다.

4. **블로커 반영**: 어제 노트의 블로커 중 해결 안 된 것, 그리고 `tasks`/`00_Inbox`에서 "권한 대기/막힘/블로커" 신호가 있으면 `### 🚧 블로커`에 옮기고, 관련 개념 페이지가 있으면 `[[...]]`로 링크한다. 블로커가 있으면 frontmatter `블로커: true`.

5. **frontmatter 채우기**: `유형: daily`, `날짜`, `주차`(`gggg-[W]ww`, 예 2026-W27), `집중시간`(비워둠 — 하루 끝에 사용자가 기입), `프로젝트`(관련 일감의 프로젝트 링크).

6. **출력**: 생성/수정한 파일 경로와, 사용자가 직접 다듬어야 할 부분(오늘 할 일 우선순위, 집중시간)을 한 줄로 안내한다.

## 형식 (본문)

```
## <날짜> (요일)
### ✅ 어제 한 일
- [[...]]
### 🎯 오늘 할 일
- [[...]]
### 🚧 블로커
- ...
### 🔗 관련 일감
- [[...]]
```

## 원칙
- 사용자가 이미 쓴 내용을 지우지 않는다. 빈 곳만 채운다.
- 추측으로 "완료"라고 단정하지 말고, 이월 항목은 사용자가 판단하도록 링크만 옮긴다.
