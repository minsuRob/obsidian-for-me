# /ingest — 원본을 위키로 정제

카파시식 LLM Wiki의 핵심 파이프라인: **raw → LLM이 이해/합성 → wiki 축적**.
대상 입력: `Work/Markhub-LLMWiki/00_Inbox/`. 산출: `Work/Markhub-LLMWiki/10_Wiki/{concepts,entities,projects}/`.

## 절대 원칙
- **원본은 절대 수정/삭제하지 않는다.** `00_Inbox`의 파일 내용은 그대로 둔다.
- 정제 완료를 표시하려면 원본 맨 아래에 `> 정제됨: [[생성한 위키페이지]]` 한 줄만 덧붙일 수 있다(내용 변경 아님). 확신이 없으면 이것도 생략.

## 절차

1. **대상 선택**: 사용자가 파일을 지정하면 그 파일, 아니면 `00_Inbox`의 처리 안 된(정제됨 표시 없는) 파일들.

2. **읽고 분해**: 원본에서 아래를 식별한다.
   - **개념(concept)**: 반복되는 주제·현상·용어 → `10_Wiki/concepts/`
   - **인물/팀/회사(entity)**: 등장하는 사람·조직 → `10_Wiki/entities/`
   - **프로젝트(project)**: 진행 중인 일의 묶음 → `10_Wiki/projects/`
   - **일감(task)**: 실행할 액션 → `20_Tracking/tasks/`에 노트 제안(우선순위 추정)

3. **위키 페이지 생성/갱신**: 해당 유형 템플릿(`_templates/wiki-*.md`) 구조를 따른다.
   - frontmatter `유형`, `태그`, `출처: ["[[원본파일명]]"]`(백링크).
   - 이미 페이지가 있으면 새로 만들지 말고 내용을 **병합/보강**한다.

4. **양방향 링크**: 관련 페이지끼리 `[[...]]`로 연결한다(개념 ↔ 프로젝트 ↔ 일감). 링크가 지식의 복리를 만든다.

5. **출력**: 생성/갱신한 페이지 목록, 새로 제안한 일감, 원본과의 연결 관계를 요약.

## 원칙
- 없는 사실을 지어내지 않는다. 원본에 근거한 것만 정제한다.
- 애매하면 페이지를 남발하기보다 기존 페이지에 링크로 흡수한다.
