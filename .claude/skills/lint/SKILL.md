# /lint — Vault 검진 (wiki를 죽지 않게 유지)

기준 경로: `Work/Markhub-LLMWiki/` (이하 `WIKI/`). **읽기 전용 리포트만** 생성한다(자동 수정 금지, 제안만).

## 점검 항목

### 1. 미처리 inbox
- `WIKI/inbox/*.md` 중 `Processed: true`가 없는 파일 → 개수와 목록. (있으면 `/ingest` 권유)

### 2. 끊긴 링크 (broken `[[links]]`)
- `WIKI/**`의 모든 `[[대상]]`을 모아, 실제 파일이 존재하지 않는 링크를 찾아 (출처 파일 → 없는 대상) 목록화.

### 3. 오래된 문서 (stale)
- `wiki/**` 문서 중 `상태: 진행중`인데 `최종수정`이 14일 이상 지난 문서 → 재검토 후보.

### 4. Frontmatter 누락/불일치
- `유형`, `상태`, `최종수정` 중 빠진 문서.
- `상태`/`우선순위` 값이 규약(대기/진행중/완료/보류, 높음/중간/낮음)을 벗어난 것.

### 5. 고아 문서 (orphan)
- 어떤 문서에서도 `[[링크]]`되지 않고, 스스로도 링크가 없는 wiki 문서 → 연결 제안.

## 출력
아래 형식의 리포트를 **채팅으로** 보고(파일로 저장하지 않음, 원하면 저장 여부 질문):
```
🔎 Markhub-LLMWiki 검진 결과 (<오늘>)
- 미처리 inbox: N건  [목록]
- 끊긴 링크: N건     [출처 → 대상]
- 오래된 문서: N건   [문서 · 최종수정일]
- frontmatter 이슈: N건 [문서 · 문제]
- 고아 문서: N건     [문서]
➡️ 추천 조치: ...
```

## 규칙
- 파일을 수정하지 않는다. 발견과 제안만 한다.
- 각 항목 0건이면 "이상 없음"으로 표기.
