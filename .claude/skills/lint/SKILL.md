# /lint — 위키 건강 점검

Markhub LLM Wiki( `Work/Markhub-LLMWiki/` )가 썩지 않게 유지.

## 먼저 할 일
1. `Work/Markhub-LLMWiki/CLAUDE.md`(스키마)를 읽는다.
2. `INDEX.md`와 `wiki/` 하위 전체 페이지를 훑는다.

## 점검 항목
1. **INDEX 정합성**: `wiki/`에 있는데 INDEX에 없는 페이지 / INDEX에 있는데 파일이 없는 페이지.
2. **깨진 링크**: 본문의 `[[링크]]` 대상 파일이 실제로 존재하는지.
3. **고아 페이지**: 아무 페이지도 링크하지 않는 페이지.
4. **정체된 일감**: `상태: 진행중`인데 `갱신일`이 30일 이상 지난 페이지.
5. **모순**: 서로 다른 페이지가 상충하는 사실을 말하는 경우.

## 절차
1. 위 항목을 점검해 **리포트만 먼저** 보여준다 (문제 목록 + 제안 수정).
2. 사용자가 확인하면 수정한다.
3. `LOG.md` 맨 아래에 `## [YYYY-MM-DD] lint | 요약` 형식으로 기록을 추가한다.

## 원칙
- raw/ 원본은 점검 대상이자 **수정 금지**. 고치는 것은 wiki/·INDEX·LOG뿐.
- 자동으로 지우지 않는다. 삭제·병합은 항상 사용자 확인 후.
