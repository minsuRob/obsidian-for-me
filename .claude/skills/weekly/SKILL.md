# /weekly — 주간회의 자동 합성

한 주치 일일보고를 종합해 주간회의 노트를 만든다. 대상: `Work/Markhub-LLMWiki/20_Tracking/weekly/<주차>.md` (예: `2026-W27.md`).

## 절차

1. **주차 결정**: 오늘 기준 현재 주차를 `gggg-[W]ww`로 계산(예: 2026-W27). 사용자가 특정 주를 지정하면 그 주차.

2. **데일리 수집**: `20_Tracking/daily/`에서 frontmatter `주차`가 해당 주차인 노트를 모두 읽는다(없으면 날짜 범위로 판단). 각 노트의 "어제 한 일/오늘 할 일/블로커"를 근거로 삼는다.

3. **자동 합성** — 주간 노트의 아래 섹션을 채운다:
   - `### 📊 이번 주 성과` : 실제 완료·진전된 항목을 일감 `[[링크]]`와 함께 요약(중복 제거).
   - `### 📅 다음 주 계획` : 미완료로 이월되는 오늘 할 일 + `상태: 진행중/대기` 일감에서 도출.
   - `### ⚠️ 리스크 / 블로커` : 주중 반복된 블로커를 묶고, 관련 개념 페이지(`[[배포 권한 병목]]` 등)로 링크.

4. **KPT 초안** — 데일리 근거에서 도출하되 단정하지 않고 "제안" 톤으로:
   - Keep: 잘 굴러간 습관·성과
   - Problem: 반복된 지연·문제
   - Try: Problem을 해결할 구체 행동

5. **frontmatter**: `유형: weekly`, `주차`.

6. **출력**: 생성한 파일 경로 + 합성에 사용한 daily 개수 + 사용자가 회의 전 확인할 포인트를 안내.

## 원칙
- **근거 기반**: 모든 성과·리스크는 특정 daily/일감에서 왔음을 링크로 추적 가능하게.
- 데이터가 없는 항목은 지어내지 말고 "(이번 주 기록 없음)"으로 남긴다.
- 카파시 원칙: 흩어진 daily를 하나의 종합 페이지로 엮는다.
