### 3) 워크트리 Mac 빌드 유일성 검증 → [[Mac 빌드 워크트리 유일성 검증]]
- 대상: `.github/workflows/staging-deploy.yml` + 원격 빌드 스크립트들.
- M5 Max Mac을 원격 SSH로 많이 쓰는데, **런마다 유일성 / 중간에 트리 변경 안 됨**을 검증해야 함.
- 확인한 사실:
  - staging-deploy는 `concurrency: staging-deploy, cancel-in-progress: true` → 파이프라인은 한 번에 하나.
  - Tailscale 호스트명은 `...-${run_id}`로 런별 유일.
  - **Electron/iOS 릴리스**: 런별 유일 워크트리 `run-${RUN_ID}-${ATTEMPT}-${SHA:0:12}` + 종류별 락(electron/ios 분리) + PID 기반 stale 락 정리 → 유일성·격리 견고.
  - **Frontend validate**: **공유 고정 워크트리** `~/.maki-ci-worktrees/frontend-validate` 재사용, 매 런 `git reset --hard $GITHUB_SHA`, **락 없음**.
  - 모든 빌드가 `$GITHUB_SHA`로 reset → 빌드 대상은 트리거 커밋에 고정(중간변경 없음)은 보장.
- 남은 검증 포인트: frontend 공유 워크트리의 무-락 경쟁(취소된 원격 빌드 잔존 프로세스), 고아 워크트리 정리, index.lock 잔존.
