# 모바일 컴포저 멘션 (사람 + 일감)

## 배경
모바일 컴포저에 **사람 언급창**(@)과 **todo/note 일감 언급**을 추가한다. 목표 경험은 웹의 "2번 느낌"(= `@` 멘션 메뉴). 사람 언급은 **mobile native로 개발**.

- 대상 파일: `apps/front/features/mobile/message/components/mobile-mark/MobileMarkComposer.tsx` (+ `.ios.tsx`, `.android.tsx`)
- 공유 뿌리: [[멘션 인프라 (Lexical)]]

## 현황
- **핵심 제약**: 모바일 컴포저는 평문 React Native `TextInput`(문자열 state)이라, 노드 기반 Lexical 멘션을 그대로 못 쓴다 → 멘션 UI를 **새로 만들어야** 함.
- iOS/Android는 각각 `@expo/ui/swift-ui`, `@expo/ui/jetpack-compose` 래퍼 존재 → 네이티브 팝오버 구현 여지 있음.
- 데이터/검색 로직은 재사용 가능: `useWorkspaceMembers`(사람), `useBucketItems`·`useSearchMyBucketItemsWithSemantic`(일감), `MentionSuggestion` 타입, `toBucketSuggestion`.

## 구현 옵션 (확인 필요 ❓ — 어느 방향으로 갈지 결정 필요)
- **A. WebView(Lexical) 임베드**: 웹 멘션 코드 100% 재사용, 의미론적 검색 자동 포함. 단 번들↑·WebView 성능 부담. "native로"라는 지시와는 거리.
- **B. 완전 네이티브 팝오버**: iOS SwiftUI Picker + Android Compose BottomSheet. UX 최적·성능 좋음. iOS/Android 코드 이원화 부담. → "1번은 native로"에 정확히 부합.
- **C. 하이브리드(추천 초기안)**: RN Modal + FlatList(웹 `MentionList` 참고) + 검색 API만 연동. 빠르게 시작, 이후 B로 고도화.

## 만들 조각 (옵션 B/C 기준)
- `useMobileMentionSuggestions.ts` — `@` 감지 + 필터 + 서버검색 통합.
- `MobilePersonMentionList.tsx` / `MobileBucketMentionList.tsx` — 멤버 / TODO·NOTE 목록.
- (native) `MobileMentionPopover.ios.tsx` / `MobileMentionSheet.android.tsx`.
- 전송 직렬화: `@user`/`#task` + mentions 메타(userId/bucketItemId, position) — 기존 `useMarkComposerSubmit.ts` 확장.
